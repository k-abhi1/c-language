Difference Between Call by Value and Call by Reference in C


call by Value


  • Copies the value of an object.
  • Guarantees that changes that alter the state of the parameter will only affect the named parameter bounded by the scope of the function.
  • Simpler to implement and simpler to reason with.
    •

    call by Reference

  • Pass a pointer that contains the memory address of an object that gives access to its contents.
  • Changes that alter the state of the parameter will reflect to the contents of the passed object.
  • More difficult to keep track of changing values that happens for each time a function may be called.
